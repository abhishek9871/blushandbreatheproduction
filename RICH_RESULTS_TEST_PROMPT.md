## TESTING INSTRUCTIONS - FOLLOW EXACTLY

### Phase 1: Navigate to Google Rich Results Test
1. Go to `https://search.google.com/test/rich-results`
2. Select the "URL" tab (should be default)
3. Ensure "Google Inspection Tool smartphone" is selected as user agent

### Phase 2: For EACH URL (repeat 38 times)

#### Step 2.1: Enter URL and Test
1. Clear any previous URL in the input field
2. Paste the production URL from the list above
3. Click "TEST URL" button
4. Wait for the test to complete (may take 60-90 seconds)
5. Confirm "Crawled successfully" appears

#### Step 2.2: Document Overview Results
Record the following from the results page:
- **Test Date/Time**: (from the crawl timestamp)
- **Total Items Detected**: (e.g., "6 items detected")
- **Overall Status**: (e.g., "Some are invalid", "All valid", etc.)

#### Step 2.3: EXPAND AND INSPECT EVERY SECTION
**This is critical - do not skip any section!**

For each detected structured data type (e.g., Product snippets, Articles, Breadcrumbs, FAQ):

1. **CLICK on the section** to expand it
2. **Record the section name** and status (Valid/Invalid/Warning)
3. **Count the items** in that section
4. **If there are INVALID items**:
   - Click to expand the invalid item
   - Record the EXACT error message
   - Record what field/property is missing
   - Record any suggested fixes
5. **If there are NON-CRITICAL ISSUES**:
   - Click to expand and see what they are
   - Record each non-critical issue
   - Record what recommended property is missing
6. **If items are VALID**:
   - Still expand to see what properties are detected
   - Record the schema properties found
   - Note if there are any optional properties missing

#### Step 2.4: Check Rendered HTML (Optional but Recommended)
1. Look at the "Tested page" panel on the right
2. Click "HTML" tab to see rendered source
3. Search for `<script type="application/ld+json">` to verify structured data
4. Click "SCREENSHOT" tab to see how Google rendered the page
5. Note any rendering issues

#### Step 2.5: Document Findings
Create a detailed entry for this URL with:

## WHAT TO LOOK FOR SPECIFICALLY

### Expected Schema Types by Page Type:

#### Pillar Pages (/banned/[slug]):
- **Drug** schema - with name, description, legalStatus, warning, adverseOutcome
- **FAQPage** schema - with 6+ Question/Answer pairs
- **BreadcrumbList** schema - 4-level trail
- **ScholarlyArticle** schema - PubMed citations

#### Cluster Articles (/guide/[slug]):
- **Article** schema - with author, datePublished, dateModified
- **FAQPage** schema - with 2+ Question/Answer pairs (if present)
- **BreadcrumbList** schema - 3-level trail

#### Supplement Pages (/supplement/[slug]):
- **DietarySupplement** schema - with benefits, dosage
- **FAQPage** schema - with Question/Answer pairs
- **BreadcrumbList** schema - 4-level trail
- **Product** schema - may show invalid (expected for educational content)

### Known Issues to Investigate:

1. **Product snippets showing "Invalid"** on Drug/Supplement schemas
   - This is expected for non-commerce pages
   - Document but note it's intentional

2. **Articles schema only showing PubMed citations**
   - Check if main page content is in Article schema
   - Document what content IS being captured
   - Document what content is MISSING

3. **Non-critical issues in Articles**
   - Usually missing `image` or `thumbnailUrl`
   - Document exactly which properties are flagged

4. **FAQ questions not appearing**
   - Verify FAQ schema is present
   - Count questions detected vs questions on actual page
